开发者指南解释 PkuClaw **为什么这样设计**、**一次任务怎样跑起来**，以及开发者应该在哪一层扩展功能。

如果你只是想安装、配置并使用 PkuClaw，请转到 [使用指南](/docs/user-guide)。本部分默认你准备阅读或修改代码、runtime 文件、prompt、skill、channel 或后台 loop。

## 先理解三件事

1. **PkuClaw 是 channel + loop + agent runtime**：用户消息、飞书菜单和后台周期任务都会进入同一个 `CoreRuntime` 控制面。
2. **运行行为尽量配置化**：quick action、loop、prompt、skill catalog 和通知策略主要来自 `configs/runtime/*`，便于热加载和通过文件 diff review。
3. **Agent 不直接碰平台 API**：realtime 默认流式回复当前对话；后台通知通过 channel outbox 写入本地队列，再由 daemon 投递到飞书等 channel。

## 推荐顺序

1. [架构与产品逻辑](/docs/developer-guide/architecture)：先建立整体心智模型。
2. [请求生命周期](/docs/developer-guide/request-lifecycle)：再看一次用户消息、菜单和后台 loop 分别怎么跑。
3. [Runtime 配置系统](/docs/developer-guide/runtime-files)：理解配置文件为什么多、热加载边界在哪里。
4. [Agent 执行链路](/docs/developer-guide/agent-runtime)：看 `CoreRuntime` 如何交给 `AgentWrapper` 和 provider。
5. [Channel 与 Outbox](/docs/developer-guide/channels-and-outbox)：理解平台接入和反向通知。
6. [Loop 与通知策略](/docs/developer-guide/loops-and-notifications)：理解后台任务为什么默认静默。
7. [Skill Catalog](/docs/developer-guide/skills)：理解 task/tool 能力如何暴露给 Agent。
8. [扩展开发](/docs/developer-guide/extension-points)：按“我要加什么功能”定位改动点。
9. [开发与贡献工作流](/docs/developer-guide/development)：提交前确认边界、测试和文档同步。
