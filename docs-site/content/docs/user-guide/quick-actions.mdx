---
title: Quick Actions
description: 把常用任务做成飞书菜单或快捷入口。
---

Quick Action 是用户主动触发的一键任务。它适合放常用操作，例如：

- 查看教学网更新；
- 查看作业概览；
- 查看本周 DDL；
- 触发一个自定义 skill。

## 文件位置

```text
configs/runtime/events.json
```

## 最小结构

一个 Quick Action 至少需要这些字段：

```json
{
  "id": "weekly_deadlines",
  "enabled": true,
  "title": "查看本周 DDL",
  "description": "快速查看未来一周课程 DDL 和需要处理的事项。",
  "task": "查看未来一周课程 DDL、未完成作业和需要处理的学习事项，并给出简洁优先级建议。",
  "suggested_skills": [
    "tasks/sync-notices.md"
  ],
  "ack": "正在查看本周 DDL。"
}
```

字段含义：

| 字段 | 说明 | 写法建议 |
| --- | --- | --- |
| `id` | PkuClaw 内部事件 id，必须唯一。 | 用英文、数字、下划线，例如 `weekly_deadlines`。 |
| `enabled` | 是否启用。 | 暂时不用就设为 `false`。 |
| `title` | 展示标题。 | 给用户看的短标题。 |
| `description` | 展示描述。 | 说明点这个按钮会做什么。 |
| `task` | 触发后传给 Agent 的任务文本。 | 写清目标、范围和禁止动作。 |
| `suggested_skills` | 建议 Agent 优先阅读的 skill。 | 使用 `skills.json` 中已注册的 skill。 |
| `ack` | 渠道收到点击后的短提示。 | 一句话即可，例如“正在查看”。 |

## 当前默认 actions

仓库默认提供了几个事件：

- `course_updates`：查看教学网更新；
- `homework_overview`：查看作业概览；
- `weekly_deadlines`：查看本周 DDL。

可以直接在 `configs/runtime/events.json` 里改标题、描述、任务文本或 suggested skills。

## 绑定到 channel

渠道适配器可以把平台原始 key 映射到 PkuClaw event id：

```json
{
  "channel_mappings": {
    "feishu": {
      "menu_weekly_deadlines": "weekly_deadlines"
    }
  }
}
```

如果平台 key 本身就是 PkuClaw event id，可以直接透传。

飞书的具体配置看 [飞书 Channel 配置](/docs/user-guide/channel/feishu)。

## 写 task 的原则

好的 `task` 应该明确：

- 要查什么；
- 输出给谁看；
- 用什么语气或格式；
- 哪些动作禁止自动执行；
- 缺少登录或配置时应该怎么处理。

示例：

```text
查看当前教学网更新、课程通知、未完成作业和 DDL，并用简洁中文总结重要变化。不要提交作业；需要登录或配置时先说明缺少什么。
```

## Suggested skills

`suggested_skills` 不会强行执行 skill，但会提示 Agent 优先阅读相关说明。

常见写法：

```json
"suggested_skills": [
  "tasks/sync-notices.md"
]
```

如果你新增了自己的 skill，先按 [Skill 配置与扩展](/docs/user-guide/skills) 注册，再在 Quick Action 里引用。

## 行为边界

- quick action 创建 `source = realtime` run；
- quick action 可以提供 suggested skills；
- quick action 不使用 loop 通知规则；
- quick action 不应该自动提交作业、安装系统包、清缓存或写入私密凭据；
- channel UI-only 事件和噪声事件应留在 channel adapter 内处理。
