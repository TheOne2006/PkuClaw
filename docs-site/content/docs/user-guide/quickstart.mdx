本页只做一件事：**尽快把 PkuClaw 在本机跑起来**。

如果你想先理解每个配置项，跳到 [配置总览](/docs/user-guide/configuration)。如果你只想先启动，按本页走。

## 方式 A：让本地 Agent 帮你安装

如果你已经在本机使用 Codex、Claude Code 或类似本地 Agent，可以把下面这句话直接发给它：

```text
请阅读并执行本仓库 docs-site/content/docs/user-guide/agent-install.mdx 的步骤，在当前机器安装并启动 PkuClaw。只改本地配置文件，不提交凭据、cookie、data、notify_queue 或构建产物；遇到需要账号、密码、OTP、飞书密钥或系统包安装的步骤先停下来问我。
```

配套的 Agent 操作手册在：

```text
docs-site/content/docs/user-guide/agent-install.mdx
```

它比本页更适合 Agent 执行：里面写了检查顺序、安全边界和验证命令。

## 方式 B：手动快速开始

### 1. 克隆仓库

```bash
git clone https://github.com/TheOne2006/PkuClaw.git
cd PkuClaw
```

### 2. 准备 Python 环境

推荐使用 `uv`：

```bash
uv sync
uv run pkuclaw --help
```

如果暂时不用 `uv`，也可以使用标准 Python editable install：

```bash
python -m venv .venv
source .venv/bin/activate
python -m pip install -e .
pkuclaw --help
```

### 3. 创建本地配置

```bash
cp configs/config.example.toml configs/config.toml
cp configs/runtime/runtime.example.json configs/runtime/runtime.json
```

然后编辑这两个本地文件：

- `configs/config.toml`：启动期配置，例如飞书 app、Codex CLI 路径、数据目录。
- `configs/runtime/runtime.json`：runtime 配置，例如模型、loop、通知目标。

这两个文件默认都不应该提交。

### 4. 配置必要凭据

如果要启用飞书渠道，需要设置飞书 app secret：

```bash
export FEISHU_APP_SECRET="你的飞书 app secret"
```

`FEISHU_APP_ID`、`FEISHU_API_BASE` 也可以通过环境变量覆盖配置文件。

如果你暂时不用飞书，可以先跳过这一步，但飞书 realtime 和 daemon 启动时仍会需要有效配置。

### 5. 运行基础检查

```bash
python -m compileall pkuclaw scripts
python -m unittest discover
```

### 6. 启动入口

开发实时路径：

```bash
uv run pkuclaw realtime feishu
```

完整 daemon：

```bash
uv run pkuclaw daemon
```

两者区别：

| 命令 | 用途 |
| --- | --- |
| `uv run pkuclaw realtime feishu` | 只跑飞书实时入口，适合先测试消息收发。 |
| `uv run pkuclaw daemon` | 跑完整 daemon，包括飞书、后台 loop 和 outbox queue worker。 |
