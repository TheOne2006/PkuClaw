OpenAI API provider 当前还没有在 PkuClaw 中实现。

这一页只是预留配置入口。现在不要把 `provider = "openai-api"` 当成可用配置。

## 当前状态

| 项目 | 状态 |
| --- | --- |
| OpenAI API provider 执行器 | 未实现 |
| `configs/config.toml` schema | 未定义 |
| `configs/runtime/runtime.json` 覆盖字段 | 未定义 |
| realtime 支持 | 未实现 |
| loop 支持 | 未实现 |

## 未来可能需要的配置

真正实现后，可能需要类似这样的启动配置：

```toml
[agent]
provider = "openai-api"

[openai]
api_key_env = "OPENAI_API_KEY"
model = "..."
timeout_seconds = 1800
max_concurrent_runs = 1
```

也可能需要 runtime 覆盖：

```json
{
  "agent": {
    "provider": "openai-api",
    "model": "..."
  }
}
```

这些字段目前只是方向，不是已支持 schema。

## 实现前需要解决的问题

接入 OpenAI API 前，至少需要明确：

- 使用哪个 API 形态；
- 如何表达工具调用、文件读写和 outbox 能力；
- 如何保存会话状态；
- 如何处理 streaming、超时和失败重试；
- loop 中是否允许联网调用；
- API key 只应来自环境变量或本地私有配置，不能写进可提交文件。
