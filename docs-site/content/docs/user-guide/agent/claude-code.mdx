Claude Code provider 当前还没有在 PkuClaw 中实现。

这一页只是预留配置入口，避免以后新增 provider 时再重排使用指南。现在不要把 `provider = "claude-code"` 当成可用配置。

## 当前状态

| 项目 | 状态 |
| --- | --- |
| Claude Code provider 执行器 | 未实现 |
| `configs/config.toml` schema | 未定义 |
| `configs/runtime/runtime.json` 覆盖字段 | 未定义 |
| realtime 支持 | 未实现 |
| loop 支持 | 未实现 |

## 未来可能需要的配置

真正实现后，可能需要类似这样的启动配置：

```toml
[agent]
provider = "claude-code"

[claude_code]
bin = "claude"
model = "..."
timeout_seconds = 1800
max_concurrent_runs = 1
```

也可能需要 runtime 覆盖：

```json
{
  "agent": {
    "provider": "claude-code",
    "model": "...",
    "reasoning_effort": "..."
  }
}
```

这些字段目前只是方向，不是已支持 schema。

## 实现前需要解决的问题

接入 Claude Code 前，至少需要明确：

- 是否有稳定的非交互式执行模式；
- 如何把 stdout/stderr 或事件流转成 PkuClaw 的 AgentEvent；
- 如何保存和恢复会话；
- 如何传入工作目录、sandbox、超时和环境变量；
- loop 中是否允许使用，以及失败后如何降级；
- 如何避免把凭据、cookie、token 写入日志。

## 当前应该怎么做

现在请继续使用 [Codex](/docs/user-guide/agent/codex)。
