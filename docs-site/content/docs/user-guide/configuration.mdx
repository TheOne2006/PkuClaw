---
title: 配置总览
description: 先分清启动配置、runtime 配置和可编辑行为配置。
---

PkuClaw 的配置不是一个文件解决全部问题。先把配置分清楚，后面改起来会容易很多。

## 三类配置

| 类型 | 文件 | 主要回答什么问题 | 是否提交 |
| --- | --- | --- | --- |
| 启动配置 | `configs/config.toml` | daemon 启动时用哪个数据目录、哪个 channel、哪个 Codex CLI。 | 不提交 |
| Runtime 配置 | `configs/runtime/runtime.json` | Agent 用什么模型、loop 多久跑一次、通知默认发给谁。 | 不提交 |
| 可编辑行为配置 | `configs/runtime/events.json`、`prompts.json`、`skills.json`、`skills/**` | 快捷任务、prompt、skill catalog 怎么写。 | 可提交，但不要写私密数据 |

第一次本地运行时，从模板复制：

```bash
cp configs/config.example.toml configs/config.toml
cp configs/runtime/runtime.example.json configs/runtime/runtime.json
```

## 启动配置：`configs/config.toml`

这个文件在 daemon 启动时读取。它适合放“启动前就必须知道”的信息。

```toml
[app]
name = "PKU Course Bot"
data_dir = "data"
runtime_config_dir = "configs/runtime"
timezone = "Asia/Shanghai"

[feishu]
app_id = "cli_xxx"
app_secret_env = "FEISHU_APP_SECRET"
event_mode = "websocket"

[agent]
provider = "codex"

[codex]
bin = "codex"
sandbox = "danger-full-access"
model = "gpt-5.5"
timeout_seconds = 1800
max_concurrent_runs = 1

[notify_queue]
queue_dir = "notify_queue"
scan_interval_seconds = 5
```

常见需要你改的是：

- `[feishu].app_id`
- `FEISHU_APP_SECRET` 环境变量
- `[codex].bin`
- `[codex].model`
- `[codex].sandbox`

:::caution
不要提交真实的 `configs/config.toml`、飞书 secret、token、cookie、用户 id 或本地路径。
:::

## Runtime 配置：`configs/runtime/runtime.json`

这个文件由 runtime 热读取，适合放“运行过程中可以调整”的本地行为。

```json
{
  "schema_version": 1,
  "agent": {
    "provider": "codex",
    "mode": "fixed",
    "model": "gpt-5.5",
    "reasoning_effort": "xhigh"
  },
  "codex": {
    "sandbox": "danger-full-access",
    "timeout_seconds": 1800
  },
  "loops": [
    {
      "id": "sync_notices",
      "enabled": true,
      "interval_seconds": 900,
      "suggested_skills": [
        "tasks/sync-notices.md"
      ],
      "sink_mode": "silent"
    }
  ],
  "notifications": {
    "policy": "important_only",
    "default_channel": null,
    "default_target_type": null,
    "default_target_id": null
  }
}
```

重点字段：

- `agent`：Agent provider、模型、reasoning effort。
- `codex`：Codex sandbox、timeout。
- `loops`：后台周期任务。
- `notifications`：后台通知策略和默认目标。

`runtime.json` 是本地文件，不提交。可提交的模板是：

```text
configs/runtime/runtime.example.json
```

## 可编辑行为配置

这些文件会影响 PkuClaw “会做什么”和“怎样做”，但不应该写入私密凭据。

| 文件 | 作用 |
| --- | --- |
| `configs/runtime/events.json` | 定义 Quick Actions，也就是用户主动触发的快捷任务。 |
| `configs/runtime/prompts.json` | 定义 realtime 和 loop prompt 模板。 |
| `configs/runtime/skills.json` | Skill Catalog，告诉 Agent 有哪些 skill 可读。 |
| `configs/runtime/skills/**` | 每个 skill 的具体操作说明。 |

## 安全边界

不要提交：

- `configs/config.toml`
- `configs/runtime/runtime.json`
- 飞书 app secret
- open id、chat id、cookie、token、账号、密码、OTP
- `data/`、`logs/`、`tmp/`、`notify_queue/` 等运行产物

可以提交：

- `configs/config.example.toml`
- `configs/runtime/runtime.example.json`
- `configs/runtime/events.json`
- `configs/runtime/prompts.json`
- `configs/runtime/skills.json`
- `configs/runtime/skills/**`

前提是这些可提交文件里没有真实私密信息。
