## 为什么是 PkuClaw？

<CardGrid stagger>
  <Card title="只保留两类 run" icon="rocket">
    `realtime` 面向用户即时交互，`loop` 面向后台周期任务，边界清楚，不靠隐式意图分类。
  </Card>

  <Card title="配置就是运行界面" icon="seti:config">
    `configs/runtime/` 是可编辑的 runtime surface：能热更新，也能通过 Git diff review。
  </Card>

  <Card title="渠道解耦" icon="puzzle">
    Agent 只写入 channel-neutral outbox，由 daemon 统一解析目标并投递到飞书等渠道。
  </Card>
</CardGrid>
